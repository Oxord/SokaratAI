## Инструкции

1. **Планирование** — войди в план-режим (EnterPlanMode). Исследуй кодовую базу, найди существующие паттерны и утилиты которые можно переиспользовать. Составь детальный план реализации в файле плана.

2. **После одобрения плана** — реализуй план: создай новую feature-ветку, используй Agent tool с `model: sonnet` для написания кода. Разбивай работу на независимые подзадачи и запускай агентов параллельно где возможно. Пиши тесты для тестирования функционала, запускай их прежде чем закончить реализацию. 

3. **После реализации** - используя команду /commit закомить изменения

4. **Проверка** — запусти агента-ревьювера командой (/code-review). Исправь все замечания.

5. **Слияние изменений** - выполни команду /merge

6. **Сохранение результата** - запиши, что ты сделал в какой-то файлик, чтобы при новой задачке не было нужды вспоимнать всё, что ты сделал до этого. Если ты делал часть какого-то плана, то обозначь сделанную часть выполненной. 

## Правила
- Не создавай файлы без необходимости — предпочитай редактирование существующих
- Переиспользуй существующие паттерны проекта (см. CLAUDE.md)
- Каждый новый модуль должен иметь unit-тесты
- Пиши интеграционные тесты, если этого требует задача
- Обязательно проверяй написанный тобой код!
