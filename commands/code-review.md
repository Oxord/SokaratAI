Проведи code review текущих изменений.

## Действия

1. Собери контекст:
   - `git diff` — unstaged изменения
   - `git diff --cached` — staged изменения
   - Если нет локальных изменений — `git diff main...HEAD` (все коммиты ветки)

2. Для каждого изменённого файла проверь:
   - **Корректность**: логические ошибки, edge cases, race conditions
   - **Безопасность**: инъекции, утечки данных, OWASP top 10
   - **Паттерны проекта**: соответствие архитектуре (см. CLAUDE.md), переиспользование существующих утилит
   - **Типизация**: правильные type hints, совместимость с mypy strict
   - **Тесты**: покрыты ли новые пути кода

3. Выдай замечания в формате:
   - Критичные (блокируют мерж)
   - Рекомендации (стоит исправить)
   - Нит (мелочи)

Привязывай каждое замечание к конкретному файлу и строке.
