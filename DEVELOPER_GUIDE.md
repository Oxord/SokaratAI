# Sokrat — Developer Guide

> AI-тренажёр собеседований | Python + LangGraph + Chainlit + OpenRouter

---

## Контекст продукта

**Sokrat** — диалоговый AI-ассистент для подготовки к собеседованиям. Пользователь выбирает целевую роль и уровень — AI берёт роль интервьюера, задаёт релевантные вопросы, анализирует каждый ответ и по итогу формирует отчёт с оценкой и рекомендациями.

### Целевая аудитория
- Junior-специалисты, проходящие первые собеседования
- Люди, меняющие профессию или карьерный трек
- Студенты, выходящие на рынок труда
- Опытные специалисты, давно не проходившие интервью

### Основной сценарий
```
Пользователь выбирает роль и уровень
    → AI ведёт адаптивное mock-интервью (от 6 до 20 вопросов — длина подбирается под уровень и список навыков)
        → После каждого ответа — мгновенный фидбэк
            → По итогу — отчёт с оценкой и рекомендациями
                → Сессия сохраняется в JSON
```

### Типы интервью
| Тип | Описание |
|---|---|
| Техническое | Знание стека, инструментов, фреймворков |
| HR / Поведенческое | STAR-метод, мотивация, конфликты |
| Смешанное | Комбинация технических и HR-вопросов |

> **Почему OpenRouter?** Единый API-ключ даёт доступ к десяткам моделей (Claude, GPT, Gemini, Mistral и др.) без отдельных аккаунтов у каждого провайдера. OpenRouter совместим с форматом OpenAI API — используется `langchain-openai` с кастомным `base_url`.

## Подключение OpenRouter

OpenRouter принимает запросы в формате OpenAI API, поэтому специальный SDK не нужен.

**Переменные окружения (`.env`):**
```
OPENROUTER_API_KEY=sk-or-...
OPENROUTER_BASE_URL=https://openrouter.ai/api/v1
MODEL_NAME=anthropic/claude-sonnet-4
```

**`.env.example`** (коммитить в репозиторий):
```
OPENROUTER_API_KEY=your_openrouter_api_key_here
OPENROUTER_BASE_URL=https://openrouter.ai/api/v1
MODEL_NAME=anthropic/claude-sonnet-4
```

Получить ключ: [openrouter.ai/keys](https://openrouter.ai/keys)

## Требования LangGraph

Граф диалога строится на `StateGraph` и должен включать:

- **State** — типизированный `TypedDict` со всеми данными сессии: роль, уровень, тип интервью, список вопросов, ответы с оценками, текущий шаг
- **Узлы (Nodes):** `greeting`, `collect_context`, `interview`, `analyze_answer`, `summary`, `save_session`
- **Условные переходы (Conditional Edges):**
  - из `collect_context` → повтор, если контекст не заполнен; переход к `interview`, если заполнен
  - из `analyze_answer` → следующий `interview`, если вопросы остались; переход к `summary`, если интервью завершено
- **Сохранение контекста** — все данные диалога хранятся исключительно в State, не в глобальных переменных

---

## Требования Chainlit

- Чат-интерфейс как основной способ взаимодействия
- **Кнопки (`cl.Action`)** для выбора роли, уровня и типа интервью — пользователь не вводит их вручную
- **Шаги (`cl.Step`)** для отображения промежуточных операций (анализ ответа, генерация отчёта)
- История диалога сохраняется в рамках сессии
- Fallback-сообщение при некорректном вводе или ошибке API

---

## Требования к LLM

- Все обращения к модели идут через OpenRouter (`base_url = https://openrouter.ai/api/v1`)
- Модель по умолчанию: `anthropic/claude-sonnet-4` — указывается в `.env`, не хардкодится
- LLM используется в трёх узлах:
  - `interview` — генерация релевантного вопроса по роли и уровню
  - `analyze_answer` — анализ ответа, возвращает структурированный JSON (оценка, вердикт, фидбэк)
  - `summary` — итоговый отчёт по всей сессии
- Смена модели производится только через `.env` без изменения кода

---

## Требования к хранению данных

- Реальная БД и внешние API не используются
- Каждая завершённая сессия сохраняется в `data/sessions/<session_id>.json`
- Банк резервных вопросов хранится в `data/questions.json` (mock-данные по ролям и уровням)
- `data/sessions/` добавляется в `.gitignore`

## Структура проекта

```
sokrat/
├── app.py                  # Точка входа Chainlit
├── graph.py                # LangGraph StateGraph — узлы и переходы
├── nodes/
│   ├── greeting.py         # Приветствие
│   ├── collect_context.py  # Сбор роли, уровня, типа интервью
│   ├── interview.py        # Генерация вопроса через LLM
│   ├── analyzer.py         # Анализ ответа через LLM
│   └── summary.py          # Итоговый отчёт + сохранение сессии
├── prompts/
│   ├── interviewer.txt     # Системный промпт интервьюера
│   ├── analyzer.txt        # Промпт анализа ответа
│   └── summary.txt         # Промпт финального отчёта
├── data/
│   ├── questions.json      # Банк вопросов по ролям (fallback)
│   └── sessions/           # Сохранённые сессии (в .gitignore)
├── utils/
│   └── validators.py       # Валидация пользовательского ввода
├── .env                    # API-ключи (в .gitignore)
├── .env.example            # Шаблон переменных окружения
├── requirements.txt
└── README.md
```

## Обязательный функционал (по требованиям хакатона)

| Требование | Реализация |
|---|---|
| Приветствие и объяснение возможностей | `greeting` узел + первое сообщение в `app.py` |
| Пошаговый диалог | `StateGraph` с последовательными узлами |
| Валидация ввода | `utils/validators.py` + условные переходы |
| Обработка некорректных запросов | Fallback в узлах + повтор шага |
| Итоговый результат | `summary` узел — отчёт с оценкой |
| Сохранение сессии | `save_session` → `data/sessions/*.json` |
| Работа через Chainlit | `app.py` — весь UI на Chainlit |

## Бонусный функционал

| Возможность | Реализация |
|---|---|
| Интерактивные элементы | `cl.Action` — кнопки выбора роли/уровня/типа |
| Mock-данные | `data/questions.json` — резервный банк вопросов |
| Fallback-сценарии | При ошибке API — вопрос из банка mock-данных |
